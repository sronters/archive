title: Архитектура
description: Архитектура обработки прайс-листов
---

MedArchive сделан как модульный монолит с отдельно масштабируемыми worker-процессами.

```text
Административная панель / интеграции
  -> FastAPI /api/v1
  -> PostgreSQL
  -> хранилище исходных файлов
  -> outbox + очередь задач
  -> адаптеры парсеров
  -> сопоставление и валидация
  -> очередь ручной верификации
  -> версии цен
  -> экспорт и вебхуки
```

## Основной поток

1. Пользователь загружает ZIP или отдельные прайс-листы.
2. Система определяет тип файла и безопасно распаковывает архив.
3. Исходники сохраняются в неизменяемом хранилище.
4. Документы уходят в обработку по формату.
5. Извлеченные строки нормализуются и сопоставляются со справочником услуг.
6. Уверенные строки принимаются автоматически.
7. Спорные строки попадают в очередь ручной верификации.
8. Проверенные цены публикуются как неизменяемые версии.
9. Результаты доступны через API, экспорт и подписанные вебхуки.

## Поддерживаемые входные данные

- PDF с текстовым слоем
- сканированный PDF через OCR-адаптер
- DOCX с таблицами
- XLSX с несколькими листами
- legacy XLS

## Надежность

- idempotency key для повторных загрузок
- сохранение исходных файлов
- отдельные запуски обработки
- transactional outbox
- аудит решений оператора
- role-based access control
- метрики и runbooks
